# 环境搭建

> [!NOTE]
> 目标：以最简成本完成本地可运行的开发环境，本地开发与部署均基于云托管服务（Supabase PostgreSQL、Upstash Redis、MongoDB Atlas、Vercel 前端、Render 后端）。

## Node 与包管理

- 使用 .nvmrc 指定 Node 24 版本，建议安装 nvm 并在进入仓库时自动切换版本
- 包管理采用 pnpm（workspace + filter），避免重复安装与加速依赖解析

```bash
# 安装 pnpm（任选其一）
corepack enable && corepack prepare pnpm@latest --activate
# 或
npm i -g pnpm

# 在仓库根安装依赖
pnpm i
```

## 数据库与依赖

- Postgres 统一由 Supabase 管理；本地开发使用 Supabase CLI 完成 link/pull/reset，迁移文件在仓库 [/supabase](file:///Users/bytedance/project/CSISP/supabase) 目录。
- Redis：统一接入 Upstash（REST API 模式）。
- Mongo：统一接入 MongoDB Atlas。

### 初始化脚本（优先）

```bash
# 在项目根执行（按操作系统选择），完成环境检查与基础指引
bash ./init_mac.sh
bash ./init_linux.sh
./init_windows.bat

# Supabase 链路（需已配置 Infisical SUPABASE_*）
pnpm run supabase:link:stag
pnpm run supabase:pull:stag
```

## IDL（契约生成）

- IDL 文件与生成流程在“基础设施与基建 / IDL（Thrift）”中说明，生成的类型在多项目间复用
- 项目使用 `@creditkarma/thrift-typescript` + `tsc` 统一生成类型，无需安装本机 Apache Thrift CLI
- 可结合根脚本的 `idl:gen` 在启动前生成契约类型（详见“快速开始”）

## 编辑器与格式

- 推荐 VS Code，启用“保存时格式化”
- 统一 EditorConfig、ESLint、Prettier 规则；前后端遵循一致的代码风格与类型约束
