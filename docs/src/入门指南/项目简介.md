# 项目简介

## 项目定位与目标

- CSISP（Computer Science Integrated Service Platform）是由 LZStarV 独立开发的 SCNU 计算机学院综合服务平台（非正式，目前仅作个人学习）。
- 本项目的作用是为了给学院的通知管理、作业管理等模块进行统一的接口设计与实现，将原先基本只能通过微信等不统一的途径进行的操作，通过本平台进行统一管理。
- 项目以开发者友好与可扩展为核心，强调清晰的模块边界、统一的接口约定与可复用的基础设施。
- 文档中心服务于开发者与 Vibe Coding，帮助快速理解架构与模块实践，并顺利完成本地启动与变更。

## 设计原则与范围

- **接口统一**：全项目采用 OpenRPC/JSON-RPC 范式进行接口设计与实现（各子项目的契约说明与静态示例在模块详解中展开）。
- **工程治理**：前后端统一的错误分类与映射、类型约束与代码风格、可访问性与性能指标；后端以中间件栈实现鉴权与限流，前端以状态与路由规范保障一致体验。
- **基建优先**：数据库（PostgreSQL）、Redis 与 Mongo 的客户端封装与健康检查，作为跨项目的稳定支撑。
- **文档约定**：采用 VitePress，目录结构统一为 index.md + src/\*，侧边栏与导航在 .vitepress/config.ts 中集中配置。

## 技术栈概览

- **后端集成服务**：NestJS（TypeScript），面向核心业务的后端编排与数据访问。
- **BFF 聚合层**：Koa 中间件栈，提供前端场景的接口聚合与错误映射。
- **后台系统**：Next.js App Router + Ant Design，JSON-RPC 统一入口与后台模块管理。
- **前端应用**：Vue3 + Vite（frontend-admin、frontend-portal），配合 BFF 或后台接口进行交互。
- **基础设施**：PostgreSQL（只读为主）、Redis（缓存与限流）、Mongo（作业、通知内容/日志/i18n）。
- **IDL 与契约**：Thrift IDL 生成流程与类型复用，统一契约在多项目间共享。
- **数据库迁移与类型生成**：提供 Migration 脚本与类型生成脚本，保障结构演进与类型一致性。

## 接口范式：OpenRPC/JSON-RPC

- 项目统一以 OpenRPC/JSON-RPC 约定进行接口设计，前后端围绕契约进行协作与校验。
- 以 backoffice 为例，路由采用 `POST /api/backoffice/:domain/:action`，响应遵循 JSON-RPC 2.0，错误映射与头部约定明确；详细规范见 [JSON-RPC 路由与响应规范](/src/模块详解/backoffice/JSON-RPC路由与响应规范)。
- 具体静态示例 JSON 与差异说明在各子项目的模块详解页中提供，便于 Vibe Coding 快速对齐接口格式。
