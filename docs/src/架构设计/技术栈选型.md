# 技术栈选型

> [!NOTE]
> 本项目统一采用 TypeScript 与单一的 OpenRPC/JSON-RPC 接口范式；`method` 由路由标识（`/api/[project?]/[subproject?]/:domain/:action`），不在请求体中使用 `method` 字段。

## 语言与框架

| 模块                   | 框架与组件                                                                 | 路由/页面                                   | 主要用途                       | 版本示例                         |
| ---------------------- | -------------------------------------------------------------------------- | ------------------------------------------- | ------------------------------ | -------------------------------- |
| `backend-integrated`   | `TypeScript`、`NestJS`、`supabase-js`（packages/supabase-sdk）、`mongoose` | `POST /api/:domain/:action`                 | 后端集成与数据访问、中间件治理 | TS^5.9、Nest^10、supabase-js^2.x |
| `bff`                  | `TypeScript`、`Koa`、`@koa/router`、`zod`、`@csisp/redis-sdk`（Upstash）   | `POST /api/bff/:subproject/:domain/:action` | 聚合编排、错误映射、限流与路由 | Koa^3.1、router^14、zod^3.23     |
| `backoffice`（待开发） | 待定                                                                       | 待定                                        | 后台管理 SPA（规划中）         | 待定                             |
| `frontend-admin`       | `TypeScript`、`Vue3`、`Vite`、`Pinia`、`Vue Router`、`Naive UI`、`ECharts` | 前端路由与页面                              | 中台前端、表格/图表与交互      | vue^3.5、vite^5.4、pinia^2.1     |

## 关键技术补充说明

- Thrift IDL 与类型共享：IDL 由 `infra/idl` 统一维护，`idl:gen` 生成后在 `bff/backend-integrated` 复用，避免类型分歧
- Supabase 迁移：以 `/supabase/migrations` 作为唯一事实来源；通过 supabase-js + supabase-sdk 进行读写
- Redis 接入：bff 与 backend-integrated 通过 `@csisp/redis-sdk` 统一接入 Upstash，并在拦截器/中间件层实现固定窗口限流（INCR+EX）
- Node 版本建议：统一使用 Node 20+（supabase-js 对 Node 18 已提示弃用）
- 日志栈：Pino 统一格式，配合 `pino-pretty` 提升开发可读性；生产可接入集中式日志平台
- Monorepo 工作流：以 Turbo 驱动 `dev:*` 与 `build`；pnpm workspace 保证依赖与脚本在多模块的一致性
